## Week 2 · Client + 集成

### 2.1 `pkg/hubclient/conn`

- [ ] `session.go` — 单 TCP：`dial`、send/recv loop、系统帧与业务帧分流
- [ ] AUTH → 等 ACK → SUB（按 `RegisterHandler` 预注册 cmd）→ Ready
- [ ] KPALIVE 30s；连续 3 次无 ACK 则断线重连
- [ ] `pool.go`（可选 M2 末）— 多连接扇出，对齐必嗨 `RTMQ_MSGQ_NUM=10`

### 2.2 `pkg/hubclient` 对外 API

- [ ] `New(cfg *Config, opts ...Option) (*Client, error)`
- [ ] `Start(ctx context.Context) error` / `Close() error`
- [ ] `RegisterHandler(cmd uint32, h Handler)` — 连接前注册；cmd=0 为默认 handler
- [ ] `AsyncSend(cmd, destNid uint32, payload []byte) error` — 线程安全
- [ ] `Ready() <-chan struct{}` 或 `WaitReady(ctx)` — 供 L3 等待 AUTH+SUB 完成
- [ ] `Option`：`WithLogger(slog.Logger)`、`WithOnReconnect(func())`

### 2.3 集成测试

- [ ] `test/integration/unicast_test.go`（`//go:build integration`）
  - 依赖 Docker：hi-im-hub + 2 个 hubclient（FORWARD consumer + BACKEND producer）
  - 对齐 beehive `rtmq-bench -mode unicast`
  - producer `AsyncSend(cmd, destNid, imFrame)` → consumer handler 收到 1 帧
- [ ] `test/integration/publish_test.go`（可选 M2 末）
  - FORWARD publish → BACKEND SUB 收到

### 2.4 文档与发版

- [ ] 核对 [技术设计文档.md](技术设计文档.md)
- [ ] 打 tag **`v0.1.0`**
- [ ] hi-im 主仓 M3 Compose 引用本 tag

## 依赖关系

| 上游 | 需要 |
|------|------|
| **hi-im-core** | Hub 进程 `:28888` / `:28889` |
| **hi-im-api** | M2 单测拼 IM 帧；L3 业务拼包（hubclient 运行时仅传 `[]byte`） |

| 下游 | 使用方式 |
|------|----------|
| **hi-im-gateway** | FORWARD 平面；SUB 下行 cmd |
| **hi-im-msgsvr / chatroom / usrsvr** | BACKEND 平面 |
| **hi-im-roomfanout** | BACKEND；Kafka 消费后 AsyncSend |

## 与 beehive 对照迁移

| 必嗨 | hi-im-hubclient |
|------|-----------------|
| `rtmq.ProxyInit` | `hubclient.New` |
| `Register` | `RegisterHandler` |
| `Launch` | `Start` |
| `AsyncSend(cmd, data, len)` | `AsyncSend(cmd, destNid, payload)` — **显式 destNid** |
| `ProxyConf` | `Config` + `ConfigFromEnv` |
| beego/logs | `log/slog`（可注入） |

**刻意改进**（相对必嗨 Go Proxy）：

1. AUTH 成功后再 SUB（消除 auth/sub 竞态）
2. `destNid` 显式参数，文档化 wire.nid 语义
3. `context.Context` 优雅退出
4. integration test 纳入 CI（可选 nightly）
